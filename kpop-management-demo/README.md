# K-pop 模拟经营 Demo 原型

这是 `[[游戏设计文档]]` 的浏览器可玩原型，用来验证 MVP 核心循环，不是最终 Unity 工程。

## 怎么运行

双击打开：

```text
kpop-management-demo/index.html
```

不需要安装 Node、Unity 或其他依赖。

## 当前已实现

- 月度循环：排 4 周行程单 -> 热搜事件 -> 月度结算 -> 下个月。
- 行程单系统：每周选择行程类型，并调整强度、预算。
- 行程类型：声乐、舞蹈、Rap、形体、内容拍摄、社媒宣传、回归制作、休整。
- 资源系统：资金、粉丝、热度、档期、压力。
- 热搜榜单：每月 5 条话题，展示标签、趋势、热度和经营建议。
- 买热搜：每月最多买 1 条，花钱放大话题，并给匹配行程额外加成。
- 热搜联动：热搜主题会改变匹配训练、内容、回归和休整行程的月报结果。
- 回归系统：单曲/迷你专、Dance/Ballad/R&B、低价/正常定价，创建项目后用“回归制作”行程推进进度。
- 事件系统：每月 1 条事件，玩家选择处理方式。
- 月报系统：展示本月热搜榜、世界新闻、行程联动结果、事件、回归进度和收益。
- 本地存档：使用浏览器 `localStorage`。

## 文件说明

| 文件 | 用途 |
| --- | --- |
| `index.html` | 页面结构，相当于 Unity 的 UI 线框 |
| `styles.css` | 视觉样式和响应式布局 |
| `game.js` | 当前行程单版核心玩法逻辑 |
| `legacy-button-prototype.js` | 上一版按钮式原型，保留作对照，不被 `index.html` 加载 |
| `assets/stage.svg` | 舞台和练习室视觉资产 |

## 设计注意

Demo 已改成行程单 + 热搜榜规则：每月 4 个周档期，每个档期选择一个行程；强度影响收益和压力，预算影响额外收益和成本。热搜榜是本月市场信号，榜单匹配行程会改变月报反馈并给小加成，买热搜会有更大的本月加成。每月报告会同时输出本月热搜榜快照和世界新闻，方便判断下个月的发展方向。压力过高会增加负面事件风险，休整用于降压。
